# Product Principles

## 1. Simplicity over technology

Toujours privilégier la solution la plus simple répondant au besoin utilisateur.

## 2. Reduce cognitive load

Chaque fonctionnalité doit supprimer un effort cognitif pour les joueurs.

## 3. One command = one intention

Une commande vocale ne doit jamais avoir plusieurs significations.

## 4. Every feature must be testable on a real court

Aucune fonctionnalité ne doit être développée sans pouvoir être testée sur un vrai terrain.

## 5. One hypothesis = one field test

Chaque évolution doit permettre de valider une hypothèse produit.

## 6. Components must remain replaceable

Le moteur de score, l'interface, le matériel et le moteur vocal doivent rester indépendants.

## 7. User value first

Aucune fonctionnalité ne doit être développée si elle n'apporte pas une valeur directe au joueur pendant un match.

## 8. Product promise

"You play. The system remembers."
